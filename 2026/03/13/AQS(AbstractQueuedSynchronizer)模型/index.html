从源码浅谈ReentrantLock的底层

AQS(AbstractQueuedSynchronizer)模型

接下来我们分别解释上面出现的名词：

state: 锁的计数器，0代表没有线程占有此锁，1代表此锁已被某个线程获取。当state >= 2时说明出现了重入。

exclusiveOuwenerThread: 锁的拥有者，记录了哪个线程拥有这个锁。

CLH(Craig-Landin-Hagersten Lock Queue)队列: 一种 FIFO 的自旋等待队列 ，由双向链表实现，AQS提供。

知道了大体框架之后我们来看看ReentrantLock是怎么实现的。

第一步：尝试释放锁 (tryRelease)

这是由 ReentrantLock 的内部类 Sync 实现的具体逻辑。它的任务是修改 state 变量。

重入锁的处理: 如果 state 是 3（重入过两次），unlock 一次后 state 变成 2，此时 free 为 false，锁依然被当前线程持有。只有当 state 减到 0 时，锁才算真正释放。

第二步：唤醒等待线程 (unparkSuccessor)

如果 tryRelease 返回了 true，AQS 就会负责去叫醒在队列里排队的线程。

线程的几种状态:

状态含义
0默认
WAITING线程正在 park
CANCELLED线程取消
SIGNAL需要被唤醒
